# Reflection - OS Lab

The hardest concept in this lab was understanding concurrency, especially race conditions in multithreading. It was difficult to understand how two threads can modify the same variable and produce different results each time.

What clicked unexpectedly was how system calls and tools like gdb and valgrind reveal what is actually happening inside the program. Seeing segmentation faults and memory leaks made the concepts very real.

Later, I would like to explore building a small shell and working more with IPC mechanisms like pipes and signals in deeper projects.
